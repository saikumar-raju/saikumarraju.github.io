AWS Cloud Security Audit — Case Study

Review of tenancy, IAM, S3, networking and logging.

Scope

  • AWS account configuration review
  • IAM policies and roles
  • S3 bucket permissions and encryption
  • Security Group & VPC checks

Methodology & Tools

Configuration review, permission analysis, policy simulation, and manual validation. Tools: AWS CLI, Prowler, ScoutSuite, custom policy checks.

Findings

  1. Over-privileged IAM roles: Several roles had wildcard actions well beyond required scope.
  2. Public S3 bucket: Sensitive backup exposed due to misconfigured ACLs.
  3. Missing MFA enforcement: Critical console accounts lacked MFA policy.

Business Impact

High risk of data exfiltration and lateral AWS account takeover. Remediation decreased exposure and tightened identity controls.

Remediation

  • Apply least-privilege for IAM roles and apply role-per-task policies.
  • Restrict S3 access and enable bucket-level encryption.
  • Enforce MFA and enable CloudTrail+GuardDuty monitoring.
